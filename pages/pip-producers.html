Pip producers is a blog site for currency and day traders.

Pip producers is built on the MERN stack architecture. MongoDB, Express, React, Node js

Initial Setup

If you are not running the application locally you can skip to "Inside the Application"


Setup prerequisits:

  • Internet connection
  • Node js and NPM (or Yarn) installed
  • Terminal access
  • Basic knowledge of each item

1) The project can be cloned directly from the git repository or downloaded.

2) Run the following scripts from the root directory to install dependencies:

        $ npm install
        $ npm run client-install
        

        ***the scripts above will work with yarn

3) To connect to a database the URI should be placed within a file called default.json and placed inside theconfig folder of the root directory. The project will work with any Mongo shell i.e. RoboMongo or mLab

Click here for RoboMongo

Click here for MongoAtlas

Input the information that corresponds to your database. The database can be named anything as long as the DB URI contains the correct title

Inside of default.json

"mongodb+srv://ursamajor:A8pV4LVlRyqgXXxu@cluster0.kng3h.mongodb.net/pip-producers?retryWrites=true&w=majority" 
        {
            'mongoURI": "mongodb://<username>:<dbpassword>@cluster0.kng3h.mongodb.net/pip-producer?retryWrites=true&w=majority",
            "JWT_SECRET": "<secret random string here>"
        }

4) Use the following scripts to run the application:

        Run server on port 5000, and client on port 3000
        $ npm run dev
        

        Only start server on port 5000
        $ npm start
        

        Only start server with watch on port 5000
        $ npm run server
        

        Only run client (port 3000) without server
        $ npm run client
        

        ***the scripts above will work with yarn

Inside the Application

The site can be found here.

Overview

Pip producers uses a roles based authorization method. A user's role is held by their web token. Their are 3 roles: user, contributor and admin. A users role determines what CRUD (create, read, update, destroy) operations they are allowed to perform. New users will automatically be assigned the role of user upon signing up for the site.

  • Users: can read published blogs
  • Contributors: can read published blogs and publish blogs. They can also save drafts and read all drafts. Contributors are able to edit and delete published blogs and drafts they create.
  • Admins: can read and write published/drafts. They can also edit or delete any published or save draft on the site. Admins can access the users page and elevate users to contributors or admins.

Once an admin is created it cannot be revoked at this time

Responsiveness

This site is fully responsive and should work on any number of different screens. The main breakpoints are 768px and 1300px. Note: this site does not contain a window.onResize handler so pages must be refreshed for layout settings to take place after a resize event.

Creating Blogs

To create a blog use the 'Create' button on the navbar. The data model for a blog contains 6 elements.

  • Title
  • Author
  • Date
  • Cover image*
  • Blog content
  • Publish option**

*Currently the blog supports one file upload per blog.

  • Click "Choose A Cover Image".
  • Select an image (.png, .jpg, .jpeg, .svg) files.
  • Click the upload button. A thumbnail of the image should appear below.

**Publish option is set to draft by default. If you wish the publish the blog simply select 'publish' on the 'publish/draft' button assemby.

Reading Blogs

To find the list page click 'blog' on the left side of the nav bar. If you are logged in as a user you will see a page titled articles which will list out the blogs. The most current 4-5 Blogs also appear at the bottom of any blog. If you are logged in as a contributor or an admin you will have the option on the list page to switch between published and drafts. Contributors will be able to read drafts from other contributors, but they will not be able to edit them.

Updating Blogs

A contributor can update any blog, published or draft, they themselves created. Admins can update any blog on the site. Clicking the 'edit' button on the edit bar will bring the user to a page that looks like the create page. The difference is the information will be auto filled into the inputs. The date is the only input that doesn't auto fill and must be re-entered

Deleting Blogs

Clicking the 'delete' button on the edit bar will immediately delete the blog from the database. There currently isn't a confirmation for this process. The blog may not appear to be deleted, to update the UI the site currently requires a refresh.

Contributors can delete any blog or draft they create. Admins can delete any blog or draft.

Note: The developers at Pip Producers kindly ask users not to delete blogs they didn't create. This will help keep the site stocked with dummy blogs for future users.

Updating Users

The site currently supports updating users to 'contributor' or 'admin' status. Only admins are able to update users. Once an admin is created it cannot be undone. Currently the site does not have a way to delete users.

Navbars and Expected Behavior

Site accessibility and authorization is defined in the navbar. That is, nav links appear differently depending on user authorization level. The 'blog' button appears on the left, but all other buttons appear on the right.





Buttons will only render if they are relevant to the current role.

Losing access during a session by refreshing the page or by entering an incorrect URL is unexpected behavior and could be the result of a bug.

Currently this site does not have an alert system. Alerts are pending for future updates.
